# GitHub Pages: Reusable workflow and setup (Template)

This repository includes a reusable GitHub Actions workflow for deploying static sites to GitHub Pages.

Files added:
- .github/workflows/pages-deploy.yml — generic Pages deployment workflow
- PAGES_SETUP.md — this file (usage and notes)

How to use in other projects:
1. Copy `.github/workflows/pages-deploy.yml` into your repository.
2. If your site requires a build step, ensure `package.json` has a `build` script (or modify workflow to run your build tool).
3. The workflow runs on pushes to `main` by default. Change `branches:` in the workflow to target another branch if desired.

Notes and security:
- Creating or updating workflow files via API may require a token with the `workflow` scope. Use the GitHub web UI or `gh auth login` (browser) to push changes if you encounter a permission error.
- Do NOT commit secrets into the repo. Use GitHub Actions Secrets for sensitive data.

Customization examples:
- To deploy only the `docs/` folder, change `path: '.'` in the `Upload artifact` step to `path: 'docs'`.
- To use a static site generator (Hugo/Jekyll), add the appropriate build/install steps before uploading the artifact.
